# docs.md

## о проекте

coredvd — это эмулятор dvd-плееров с собственным 32-битным процессором pulse.  
прошивки для pulse пишутся на ассемблере и компилируются в бинарный файл.  
в будущем в проекте появятся и другие процессоры (z80, arm и т.д.),  
но сейчас основная цель — показать, как работает программируемый dvd-плеер.

## процессор pulse

pulse — risc-подобный 32-битный процессор с 16 регистрами (r0..r15).  
поддерживает флаг z (zero), который устанавливается после многих инструкций.  
стек растёт вниз, указатель стека — r14.  
программный счётчик (pc) — внутренний, не виден как регистр.

характеристики:
- 32-битные инструкции (переменной длины)
- прямая адресация памяти (до 64 кбайт озу)
- отдельная видеопамять (vram) для кадрового буфера

## система команд

все инструкции пишутся в нижнем регистре.  
аргументы разделяются запятыми.

список инструкций (опкод, синтаксис, действие):

### пересылка данных
- `mov rX, imm32` — загрузить 32-битную константу в регистр
- `movi rX, imm16` — загрузить 16-битную константу (little-endian)
- `load rX, addr32` — загрузить слово (4 байта) из озу
- `store addr32, rX` — сохранить слово в озу
- `ldrb rX, addr32` — загрузить байт
- `strb addr32, rX` — сохранить байт
- `ldrh rX, addr32` — загрузить полуслово (2 байта)
- `strh addr32, rX` — сохранить полуслово
- `push rX` — поместить регистр в стек
- `pop rX` — восстановить регистр из стека

### арифметика и логика
- `add rX, rY` — rX = rX + rY
- `sub rX, rY` — rX = rX - rY
- `mul rX, rY` — rX = rX * rY
- `div rX, rY` — rX = rX / rY (целочисленное)
- `mod rX, rY` — rX = rX % rY
- `and rX, rY` — побитовое и
- `or rX, rY` — побитовое или
- `xor rX, rY` — исключающее или
- `not rX` — побитовое отрицание
- `neg rX` — арифметическое отрицание (дополнительный код)
- `inc rX` — увеличить на 1
- `dec rX` — уменьшить на 1
- `cmp rX, rY` — сравнить (выставляет флаг z)

### сдвиги
- `lsl rX, rY` — логический сдвиг влево на rY бит
- `lsr rX, rY` — логический сдвиг вправо
- `asr rX, rY` — арифметический сдвиг вправо (знаковый)
- `ror rX, rY` — циклический сдвиг вправо

### управление потоком
- `jmp addr32` — безусловный переход
- `jz addr32` — переход, если флаг z = 1
- `jnz addr32` — переход, если флаг z = 0
- `call addr32` — вызов подпрограммы (адрес возврата в стеке)
- `ret` — возврат из подпрограммы
- `halt` — останов процессора

### системные и видео
- `print_char rX` — вывести на консоль символ из младшего байта регистра
- `load_frame` — показать текущий кадр из vob (специальная инструкция для dvd)
- `swi imm8` — системное прерывание (код 0 = выход, 1 = очистка экрана)
- `nop` — нет операции

## ассемблер

ассемблер `asm.py` преобразует текстовый файл с расширением `.asm` в бинарный `.bin`.

запуск:
```bash
python3 asm.py firmware.asm firmware.bin
```

особенности:
- метки пишутся с двоеточием, например `loop:`
- комментарии начинаются с `;`
- числа: десятичные (123) или шестнадцатеричные с префиксом 0x
- регистры: r0..r15 (строчные буквы)
- инструкции не чувствительны к регистру

пример прошивки:
```asm
start:
    mov r0, 0       ; указатель на начало vob
    mov r4, 0       ; счётчик кадров
loop:
    load_frame      ; показать кадр
    jmp loop
```

## память и ввод-вывод

- **озу** (ram): 64 кбайт, адреса 0x0000..0xffff
- **vram**: область rgb (320×240×3 байта) доступна только для чтения через графическую подсистему
- **vob-данные**: читаются процессором через регистр r0 (указатель на текущий кадр)
- **порты ввода-вывода** (in/out) пока не реализованы, но зарезервированы на будущее

## сборка и запуск

требования:
- компилятор c++17
- библиотека sfml 2.5+
- python 3.6+

сборка эмулятора:
```bash
g++ -std=c++17 main.cpp -o coredvd -lsfml-graphics -lsfml-window -lsfml-system -pthread
```

запуск:
```bash
./coredvd
```
эмулятор загружает `firmware.bin` из текущей папки и пытается открыть `VIDEO_TS/VTS_01_1.VOB`.  
если vob нет — генерируется тестовое видео.

## управление в эмуляторе

| клавиша | действие |
|---------|----------|
| m | открыть/закрыть osd-меню |
| esc | закрыть меню |
| пробел | play / pause |
| s | stop |
| ← → | предыдущая / следующая глава |
| d | включить тихую отладку (вывод каждые 1000 инструкций) |
