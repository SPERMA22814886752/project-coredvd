# docs.md (процессор z80)

## общие сведения

z80 - классический 8-битный процессор, который coredvd эмулирует для запуска прошивок.  
все прошивки для z80 должны лежать в `firmware/z80/firmware.bin` и собираться ассемблером `asm.py` из корня проекта.

## регистры и память

- **a** — аккумулятор, 8 бит  
- **b, c, d, e, h, l** — 8-битные регистры общего назначения  
- **флаги (z, s, p, c, h, n)** — обновляются после инструкций `add_a_b` и других арифметических команд  
- **pc** — 16-битный счётчик команд  
- **sp** — 16-битный указатель стека (пока не используется)  

**особенность coredvd:** внутри эмулятора есть 32-битный счётчик кадров `regs[15]`. он не доступен напрямую из z80-кода, но автоматически увеличивается при каждом вызове `load_frame` в режиме воспроизведения.

## системные соглашения

- прошивка загружается по адресу `0x0000`, выполнение начинается с `pc = 0`.
- инструкция **`load_frame`** (кодируется как `RST 18`, опкод `0xDF`)  
  - декодирует следующий кадр видео через ffmpeg  
  - применяет выбранные видеоэффекты (яркость, контраст, сканлайны, ч/б)  
  - рисует osd (время, статус play/pause, главу)  
  - увеличивает внутренний счётчик кадров на 1  
  - если видео закончилось, переключает эмулятор в состояние `DISC_MENU`
- если прошивка не вызывает `load_frame` в цикле, экран не будет обновляться (останется чёрным или с логотипом).
- переходы между главами (вперёд/назад) обрабатываются эмулятором по клавишам → / ← и не требуют вмешательства прошивки.

## поддерживаемые инструкции (те, что нужны для прошивок coredvd)

| инструкция  | опкод | описание |
|-------------|-------|-----------|
| `nop`       | 0x00  | ничего не делать |
| `ld_a n`    | 0x3E  | загрузить 8-битное число в регистр a |
| `ld_b n`    | 0x06  | загрузить 8-битное число в регистр b |
| `add_a_b`   | 0x80  | a = a + b (обновляет флаги) |
| `jp nn`     | 0xC3  | безусловный переход по 16-битному адресу |
| `load_frame`| 0xDF  | запросить кадр (псевдоинструкция, в реальном z80 это `rst 18`) |
| `halt`      | 0x76  | остановить процессор (эмулятор завершит поток) |

> **примечание:** полный набор инструкций z80 намного больше, но в coredvd используются только перечисленные. остальные (если они встретятся в прошивке) приведут к непредсказуемому поведению.

## примеры прошивок для z80

### 1. минимальная прошивка (просто крутим видео)

```asm
start:
    ld_a 0        ; инициализация не обязательна, но для порядка
    ld_b 0
loop:
    load_frame    ; показать следующий кадр
    jp loop       ; бесконечный цикл
```

### 2. прошивка с демонстрацией сложения (a увеличивается на b каждый кадр)

```asm
init:
    ld_a 10       ; a = 10
    ld_b 20       ; b = 20
loop:
    add_a_b       ; a = a + b (10+20=30, затем 50, 70...)
    load_frame    ; показать кадр
    jp loop
```

### 3. прошивка с остановкой после 100 кадров

```asm
    ld_a 100      ; счётчик кадров (но реальный счётчик всё равно внутри эмулятора, это просто пример)
loop:
    dec a         ; a-- (эта инструкция не реализована! так нельзя)
    load_frame
    jp nz, loop   ; если a != 0, то перейти
    halt
```

> **важно:** в текущей версии coredvd инструкции `dec`, `jp nz` и другие **не поддерживаются**. используйте только перечисленные в таблице выше. если нужна полноценная эмуляция z80, придётся расширить ядро.

## ассемблер z80

в корне проекта лежит скрипт `asm.py`, который умеет собирать код для z80.

**синтаксис:**
- метка: `имя:` (в отдельной строке или перед инструкцией)
- комментарии: после `;`
- числа: десятичные (`10`) или шестнадцатеричные (`0x0A`)

**пример файла `firmware.asm`:**  
```asm
; простая прошивка для z80
start:
    ld_a 0x10
    ld_b 0x20
loop:
    add_a_b
    load_frame
    jp loop
```

**сборка:**
```bash
python3 asm.py --arch=z80 firmware.asm firmware/z80/firmware.bin
```

после сборки запускай эмулятор:
```bash
./coreDVD --cpu=z80
```

## отладка

- в эмуляторе нет встроенного отладчика для z80, но можно добавить вывод в консоль при каждой инструкции (изменив `step_z80()` в исходном коде).
- если прошивка не работает — проверь, что `firmware/z80/firmware.bin` существует и собран правильным ассемблером.

## ограничения текущей реализации z80 в coredvd

- нет работы с памятью (кроме rom прошивки) — все инструкции работают только с регистрами a и b.
- нет прерываний, нет стека, нет инструкций ввода-вывода.
- `load_frame` — это костыль через `rst 18`, а не полноценный системный вызов.
- размер прошивки ограничен размером файла (теоретически до 64 КБ, но эмулятор не проверяет границы).
